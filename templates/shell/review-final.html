一、核心概念速查表

概念 一句话定义 关键公式 / 要点 难度
虚拟内存 进程地址空间的抽象，由 MMU 负责虚实转换 程序 > RAM 也能运行 ⭐⭐
有效访问时间 考虑缺页代价后的平均内存访问时间 $$EAT=(1-p)T_{mem}+pT_{disk}$$ ⭐⭐
缺页率 p 内存访问中触发缺页中断的比例 $$p=\frac{\text{faults}}{\text{accesses}}$$ ⭐
TLB 页表缓存 (Translation Lookaside Buffer) 命中率 > 99%，查表 ~1 cycle ⭐⭐
Belady 异常 FIFO 置换算法的反常行为 分配更多帧反而增加缺页 ⭐⭐⭐
页面置换算法 内存满时选择换出哪个页面的策略 FIFO / LRU / Clock ⭐⭐
多级页表 用层级结构减少页表对内存的占用 x86-64: 4 级，每级 512 项 ⭐⭐⭐
Copy-on-Write fork() 时父子进程共享页，写时复制 节省内存，延迟拷贝 ⭐⭐

有效访问时间

$$EAT = (1-p) \times T_{\text{mem}} + p \times T_{\text{disk}}$$
p = 缺页率
Tmem ~ 100ns
Tdisk ~ 10ms

TLB 加速下的 EAT

$$\begin{aligned} EAT &= h(T_{\text{TLB}}+T_{\text{mem}}) \\ &+ (1-h)(T_{\text{TLB}}+kT_{\text{mem}}) \end{aligned}$$
h = TLB 命中率
k = 页表级数

页表大小估算

$$\text{PT Size} = \frac{\text{VA Space}}{\text{Page Size}} \times \text{PTE Size}$$
e.g. 48-bit VA, 4KB page, 8B PTE
→ 256TB / 4KB × 8B = 512GB

✍️ 自测检验（点击问题查看答案）

Q1: 缺页率 p = 0.0001%，EAT 是多少？
$$EAT \approx 100\text{ns} + 10^{-6}\times 10\text{ms} = 110\text{ns}$$ 几乎无影响——缺页率极低时，虚拟内存的性能代价可忽略。
Q2: 为什么 LRU 实际很少直接实现？
真正的 LRU 每次内存访问都需更新链表（开销大），且需硬件时间戳支持。OS 通常用 Clock 算法近似：1 个访问位 + 时钟指针扫描。
Q3: TLB shootdown 是什么？
多核 CPU 中，一个核修改了页表后必须通知其他核刷新 TLB。Linux 通过 IPI (核间中断) 实现，又称 TLB flush。
Q4: 什么场景下 Belady 异常会发生？
仅在使用 FIFO 置换算法时可能出现。LRU 和 Clock 都属于栈算法 (stack algorithm)，不会出现 Belady 异常。
