📐 关键公式
$$EAT = (1-p)T_{mem} + p T_{disk}$$
虚拟内存

操作系统为每个进程提供的一个独立的、连续的地址空间抽象。进程看到的地址（虚拟地址）不等于物理内存地址。

有效访问时间

$$EAT = (1-p) \times T_{\text{mem}} + p \times T_{\text{disk}}$$
p = 缺页率  |  Tmem = 内存访问时间 (~100ns)  |  Tdisk = 磁盘访问时间 (~10ms)

缺页率从 0.1% 降到 0.01%，EAT 从 10μs 降到 1μs——100 倍差距。这就是为什么页面置换算法这么重要。

没有虚拟内存会怎样？

❌ 进程 A 的 bug 可能覆盖进程 B 的内存

✅ 虚拟内存提供隔离——每个进程看到自己的地址空间

具体例子

游戏加载 60GB 纹理，但你的电脑只有 16GB RAM。操作系统用虚拟内存让游戏"以为"有 60GB——实际只加载当前需要的页面到物理内存。

CPU 发出虚拟地址

进程执行 mov eax, [0x7fff1234]

MMU 查 TLB

快表命中 → 直接拿到物理地址（~1 cycle）

TLB miss → 查页表

MMU 遍历多级页表，找到物理页框号

Page fault → 缺页中断

如果页面不在内存，OS 从磁盘换入，更新页表

CPU → [虚拟地址] → MMU → [查TLB]
                  ↓ miss
               [页表遍历]
                  ↓
            [物理地址] → RAM

📋 本章总结

概念定义关键公式
虚拟内存进程地址空间的抽象—
有效访问时间平均内存访问时间$$EAT = (1-p)T_{mem} + pT_{disk}$$
缺页率访问的页面不在内存的概率$$p = \frac{\text{page faults}}{\text{total accesses}}$$
Q: 没有虚拟内存会怎样？
进程直接操作物理地址：① 进程间无隔离（安全问题）② 必须全部加载到内存（浪费）③ 碎片化严重
